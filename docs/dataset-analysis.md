# 데이터셋 비교 분석

작성일: 2026-06-18  
분석 대상: `data/dataset`, `data/dataset-v1`, `data/dataset-v2`

## 1. 개요

본 프로젝트에서는 VitalDB 원본 `.vital` 파일을 기반으로 세 가지 버전의 PPG-혈압 데이터셋을 구축하였다.
각 버전은 전처리 방식과 품질 필터링 기준이 다르며, 결과적으로 규모·분포·레이블 산출 방식에서 뚜렷한 차이를 보인다.

| 항목          | dataset (기본)                               | dataset-v1                                         | dataset-v2                                         |
| ------------- | -------------------------------------------- | -------------------------------------------------- | -------------------------------------------------- |
| 구축 스크립트 | `construct-dataset.py`                       | `construct-dataset-v1.py`                          | `construct-dataset-v2.py`                          |
| 입력          | `.vital` 원본 파일                           | `data/dataset` NPZ (PPG만)                         | `.vital` 원본 파일                                 |
| 품질 필터     | NaN·Inf 제거, 생리적 범위 검사               | + PPG Welch power_ratio ≥ 0.6                      | + ABP·PPG 9단계 정제 룰                            |
| 레이블 출처   | `Solar8000/ART_SBP·DBP` 1 Hz 평균            | `Solar8000/ART_SBP·DBP` 1 Hz 평균                  | ABP 파형 peak/foot 직접 추출                       |
| ABP 트랙 사용 | 레이블 생성에 미사용                         | 레이블 생성에 미사용                               | 필수 (Peak/Foot 검출 + 레이블)                     |
| 관련 문서     | [construct-dataset.md](construct-dataset.md) | [construct-dataset-v1.md](construct-dataset-v1.md) | [construct-dataset-v2.md](construct-dataset-v2.md) |

## 2. 전체 데이터셋 규모 비교

### 2.1 세그먼트 및 케이스 수

| **val**   | 977,299 segs / 339 cases         | 962,633 segs / 339 cases         | 4,787 segs / 242 cases        |

> v2의 비율 편차는 케이스별 생존 세그먼트 수가 불균등하여 split 간 비율이 설계 목표(70/10/20)에서 소폭 이탈한 결과다.

### 2.2 필터링 감소율

| 경로           | 세그먼트 수 | 감소 세그먼트  | 감소율     |
| -------------- | ----------- | -------------- | ---------- |
| dataset (기준) | 9,837,962   | —              | —          |
| → dataset-v1   | 9,687,189   | 150,773 제거   | **–1.5%**  |
| → dataset-v2   | 44,664      | 9,793,298 제거 | **–99.5%** |

dataset → v1 전환 시 power_ratio 필터는 전체 세그먼트의 1.5%만 제거한다.
반면 dataset → v2 전환 시 9단계 ABP 기반 정제 룰은 99.5%를 제거하며,
최종 생존 세그먼트 수는 dataset 대비 **약 220분의 1** 수준이다.

케이스 기준으로는 dataset-v1이 1개 케이스만 탈락한 반면,
dataset-v2는 1,084개 케이스(전체의 32.7%)가 유효 세그먼트를 하나도 확보하지 못해 제외된다.

### 3.3 주요 관찰

#### **dataset ↔ dataset-v1 비교**

두 데이터셋의 혈압 분포는 거의 동일하다. power_ratio 필터(PPG 주파수 품질)는 혈압값 자체를 기준으로 선별하지 않으므로 SBP·DBP 통계량이 사실상 변하지 않는다. 분포 그래프에서도 두 데이터셋의 곡선이 겹쳐 보인다.

#### **dataset-v2 SBP 평균 상승 (+6.5 mmHg)**

dataset-v2의 SBP 평균은 dataset보다 약 6.5 mmHg 높다. 이는 두 가지 요인이 복합된 결과다.

1. **레이블 산출 방식 차이**: dataset/v1은 `Solar8000/ART_SBP` 1 Hz 수치 트랙을 구간 평균하여 레이블을 만든다. v2는 ABP 파형에서 peak(수축기 최고점)를 직접 검출하여 평균한다. 파형 peak는 1 Hz 평균값보다 수mmHg 높게 나오는 경향이 있다.
2. **룰 3 하한 적용**: v2는 SBP ≥ 60 mmHg를 강제하므로, 저혈압 구간 세그먼트가 제거된다. 이로 인해 분포 하단이 절사(truncation)되어 평균이 상승한다.

#### **dataset-v2 SBP 범위 축소 (249.25 → 179.53 mmHg)**

룰 3(SBP ≤ 180 mmHg)과 룰 8(세그먼트 내 SBP 변동 ≤ 40 mmHg)이 분포 상단을 제거한다. 결과적으로 극단적 고혈압 세그먼트가 사라진다.

#### **dataset-v2 DBP 범위 축소 (20–150 → 40–112 mmHg)**

룰 3(DBP 40–120 mmHg)이 하한을 40 mmHg로 올린다. 저이완기혈압 구간이 제거되면서 DBP 분포의 왜도가 줄어든다.

#### **dataset-v2 SBP 분포 양봉화(bimodal)**

bp_distribution.png에서 v2 SBP 분포는 ~100 mmHg와 ~150 mmHg 부근에 두 봉우리가 생기는 양봉 형태를 보인다. dataset/v1의 단봉 분포와 대조적이다. 이는 9단계 필터가 일부 혈압 구간에서 세그먼트를 과도하게 탈락시켜 분포가 불연속적으로 변형된 것으로 판단된다. 특히 train split에서 두드러지며, split 간 분포 불일치도 관찰된다.

### 4.1 통계 요약 — train split 기준

| 통계량               | dataset | dataset-v1 | dataset-v2 |
| -------------------- | ------- | ---------- | ---------- |
| mean                 | 2,983   | 2,939      | **21**     |
| std                  | 1,525   | 1,505      | **64**     |
| min                  | 46      | 37         | **1**      |
| max                  | 14,584  | 14,502     | **912**    |

| 상위 10% 케이스 비중 | 20.3%   | 20.3%      | **61.9%**  |

### 4.2 주요 관찰

#### **dataset·v1: 준균일한 분포**

dataset과 v1은 케이스당 세그먼트 수가 대략 정규 분포에 가까운 단봉 형태를 보인다. 중간 케이스(2,700–3,000 segs)가 가장 많고, 양 끝으로 완만하게 감소한다. 상위 10% 케이스가 전체 세그먼트의 약 20%를 차지하여 집중도가 낮은 편이다.

#### **dataset-v2: 극단적인 오른쪽 꼬리 분포**

v2는 대부분의 케이스(중앙값 6 segs)가 1~10개의 세그먼트만 살아남고,
극소수 케이스(최대 912 segs)가 수백 개를 보유하는 멱함수형(power-law) 분포를 보인다.
상위 10% 케이스가 전체 세그먼트의 62%를 차지하며, 최대/중앙값 비율이 152×에 달한다.

이 집중 현상의 원인은 9단계 정제 룰이 모든 세그먼트에 동일하게 적용되기 때문이다.
수술 시간이 길고 생체신호가 안정적인 케이스(예: 장시간 마취 유지, 혈압 변동 적음)는
많은 세그먼트가 생존하지만, 수술 시간이 짧거나 혈역학적으로 불안정한 케이스는
세그먼트 대부분이 탈락한다.

## 5. 케이스 내 혈압 변동성 비교 (SD per Case)

sd_per_case.png는 케이스 내 SBP·DBP 표준편차를 케이스 단위로 정렬한 분포다.
이 지표는 교정(calibration) 없이 운용되는 혈압 추정 모델의 난이도를 반영한다.

| 통계량                      | dataset   | dataset-v1 | dataset-v2   |
| --------------------------- | --------- | ---------- | ------------ |
| SBP 케이스 내 SD (중앙값)   | 16.2 mmHg | 16.1 mmHg  | **8.8 mmHg** |
| SBP 케이스 내 SD (평균)     | 16.8 mmHg | 16.7 mmHg  | **9.0 mmHg** |
| SBP SD ≤ 5 mmHg 케이스 비율 | 0.1%      | 0.1%       | **31.0%**    |
| DBP 케이스 내 SD (중앙값)   | 8.9 mmHg  | 8.9 mmHg   | **4.6 mmHg** |
| DBP 케이스 내 SD (평균)     | 9.2 mmHg  | 9.2 mmHg   | **4.9 mmHg** |
| DBP SD ≤ 5 mmHg 케이스 비율 | 1.4%      | 1.7%       | **54.1%**    |

dataset-v2는 케이스 내 SBP 변동성이 dataset 대비 약 절반 수준(8.8 vs 16.2 mmHg)으로 낮다.
이는 룰 8(세그먼트 내 SBP 변동 ≤ 40 mmHg, DBP 변동 ≤ 20 mmHg)이 혈역학적 과도기 구간을 걸러내기 때문이다. 결과적으로 v2에 포함되는 케이스는 혈압이 상대적으로 안정된 구간에 집중된다.

v2에서 SBP SD ≤ 5 mmHg인 케이스가 31%에 달한다는 것은, 이 케이스들에서는 세그먼트 내 혈압 변동이 거의 없어 레이블의 대표성이 높다는 의미이다. 반면 dataset/v1에서 같은 조건을 만족하는 케이스는 0.1%에 불과하다.

## 6. 레이블 산출 방식 비교

| 항목                 | dataset / dataset-v1                           | dataset-v2                                           |
| -------------------- | ---------------------------------------------- | ---------------------------------------------------- |
| 레이블 신호 출처     | `Solar8000/ART_SBP`, `ART_DBP` (1 Hz 수치)     | `SNUADC/ART` ABP 파형 (500 Hz → 125 Hz)              |
| SBP 산출 방법        | 세그먼트 구간 SBP 수치 평균                    | ABP peak 검출 후 `abp_raw_seg[peaks]` 평균           |
| DBP 산출 방법        | 세그먼트 구간 DBP 수치 평균                    | ABP foot 검출 후 `abp_raw_seg[foots]` 평균           |
| 시간 정합성          | 1 Hz 수치와 PPG 파형 간 약간의 시간 오프셋     | ABP·PPG 동기화된 파형에서 직접 추출 → 시간 정합 보장 |
| 레이블 값 특성       | SBP·DBP 이동 평균에 가까운 부드러운 값         | 실제 수축기·이완기 피크 진폭 → 더 높은 SBP 경향      |
| 저혈압·고혈압 극단값 | 생리 범위(SBP 50–250, DBP 20–150) 내 모두 포함 | 룰 3으로 SBP 60–180, DBP 40–120으로 제한             |

동일 케이스에서도 v2 레이블과 dataset/v1 레이블 간에 수 mmHg 차이가 발생할 수 있다.
따라서 v2로 학습한 모델의 성능을 dataset/v1 기반 모델과 직접 수치 비교할 때는 이 차이를 감안해야 한다.

## 7. 필터링 기준 비교

### 7.1 공통 기본 필터 (세 데이터셋 모두 적용)

| 조건                           | 상세                               |
| ------------------------------ | ---------------------------------- |
| NaN·Inf 포함 PPG 세그먼트 제거 | guard-band 포함 구간 전체 검사     |

### 7.2 dataset-v1 추가 필터

| 조건                    | 방법                               | 임계값 |
| ----------------------- | ---------------------------------- | ------ |
| PPG 심박수 대역 전력 비 | Welch PSD, 0.67–3.0 Hz / 0.5–10 Hz | ≥ 0.6  |

### 7.3 dataset-v2 추가 필터 (9개 룰)

| 룰   | 조건                                  | 방법·임계값                                        |
| ---- | ------------------------------------- | -------------------------------------------------- |
| 룰 1 | 연속 동일값 검사 (flat line 감지)     | ABP·PPG 각각, 연속 횟수 ≤ 10 (80 ms @ 125 Hz)      |
| 룰 2 | FASQA Adaptive — ABP 주파수 품질 평가 | psd_low < 0.15, psd_target > 0.10, psd_high < 0.05 |
| 룰 2 | FASQA Adaptive — PPG 주파수 품질 평가 | 동일 (Min-Max 정규화 후 적용)                      |
| 룰 3 | 혈압 범위 제한                        | SBP 60–180 mmHg, DBP 40–120 mmHg                   |
| 룰 4 | 심박수 범위 제한                      | ABP·PPG 각각 30–150 bpm                            |
| 룰 5 | ABP–PPG 심박수 일치                   | 차이 ≤ 10 bpm                                      |
| 룰 6 | Peak/Foot 개수 차이                   | ABP·PPG 각각 ‖N_peak − N_foot‖ ≤ 2                 |
| 룰 7 | 최소 Peak/Foot 개수 (레이블 신뢰도)   | ABP peak ≥ 4, foot ≥ 4                             |
| 룰 8 | 세그먼트 내 혈압 변동 범위            | SBP 변동 ≤ 40 mmHg, DBP 변동 ≤ 20 mmHg             |
| 룰 9 | PPG Skewness (파형 방향성)            | foot-to-foot 사이클 평균 skewness > 0              |

## 9. 결론 및 활용 지침

### dataset (기본 버전)

- 가장 큰 규모(약 1,000만 세그먼트)로 모델 사전학습이나 대규모 실험에 적합하다.
- 레이블이 1 Hz 이동 평균 값이므로 세그먼트 내 혈압 변화가 평균으로 눌려 있다.
- 극단적 혈압값(저혈압·고혈압)을 포함하므로 분포 폭이 넓다.

### dataset-v1 (PPG 주파수 필터링)

- dataset에서 PPG 신호 품질이 낮은 세그먼트(1.5%)를 제거했으나 통계적 특성은 dataset과 거의 동일하다.
- power_ratio 단일 기준 필터는 ABP 품질, 생리적 타당성 등을 평가하지 않아 실질적인 개선 효과가 미미하다.
- dataset의 경량 대안으로 사용할 수 있으나 dataset-v2와 같은 강한 정제 효과는 기대하기 어렵다.

### dataset-v2 (9단계 ABP 기반 정제)

- 세그먼트 수가 대폭 감소(44,664개)하여 학습 데이터 부족 문제가 발생할 수 있다.
- 케이스당 세그먼트가 중앙값 6개로 극히 적어, 세그먼트가 많은 소수 케이스에 편향될 위험이 있다(상위 10% 케이스가 세그먼트의 62% 점유).
- 혈압 분포가 dataset/v1 대비 이중봉(bimodal) 형태를 보이며 split 간 분포 불일치도 관찰된다. train/val/test 분포 일관성 확인이 필요하다.
- 레이블이 ABP 파형에서 직접 추출되어 시간 정합이 보장되고 레이블 신뢰도가 높다.
- 케이스 내 혈압 변동성이 낮아(SBP SD 중앙값 8.8 mmHg) 세그먼트-레이블 정합 품질이 우수하다.
- 데이터 양보다 레이블 품질을 우선시할 때, 또는 혈압이 안정된 구간에서의 PPG-BP 관계를 집중적으로 학습할 때 유리하다.

### 모델 학습 전략 권고

1. **대규모 학습**: dataset 또는 dataset-v1을 사용한다. 두 데이터셋의 통계적 특성이 거의 동일하므로 어느 쪽을 써도 결과가 유사하다.
2. **고품질 파인튜닝**: dataset 또는 v1으로 사전학습 후 dataset-v2로 파인튜닝하면 레이블 정확도가 높은 데이터의 장점을 취할 수 있다.
3. **v2 단독 학습 시 주의**: 데이터 부족과 케이스 편중을 완화하기 위해 weighted sampling 또는 `--min-peaks` 완화(4 → 3) 등을 검토해야 한다.
4. **모델 간 공정 비교**: 서로 다른 데이터셋으로 학습한 모델을 비교할 때는 레이블 출처 차이(Solar8000 평균 vs ABP peak/foot)가 수 mmHg 오차를 유발할 수 있음을 명시해야 한다.
