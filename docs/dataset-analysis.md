# 데이터셋 비교 분석

작성일: 2026-06-22  
분석 대상: `data/dataset`, `data/dataset-v1`, `data/dataset-v2`

## 1. 개요

본 프로젝트에서는 VitalDB 원본 `.vital` 파일을 기반으로 세 가지 버전의 PPG-혈압 데이터셋을 구축하였다.
각 버전은 전처리 방식과 품질 필터링 기준이 다르며, 결과적으로 규모·분포·레이블 산출 방식에서 뚜렷한 차이를 보인다.

| 항목          | dataset (기본)                               | dataset-v1                                         | dataset-v2                                         |
| ------------- | -------------------------------------------- | -------------------------------------------------- | -------------------------------------------------- |
| 구축 스크립트 | `construct-dataset.py`                       | `construct-dataset-v1.py`                          | `construct-dataset-v2.py`                          |
| 입력          | `.vital` 원본 파일                           | `data/dataset` NPZ (PPG만)                         | `.vital` 원본 파일                                 |
| 품질 필터     | NaN·Inf 제거, 생리적 범위 검사               | + PPG Welch power_ratio ≥ 0.6                      | + ABP·PPG 9단계 정제 룰 (Rule 2 제외)              |
| 레이블 출처   | `Solar8000/ART_SBP·DBP` 1 Hz 평균            | `Solar8000/ART_SBP·DBP` 1 Hz 평균                  | ABP 파형 peak/foot 직접 추출                       |
| ABP 트랙 사용 | 레이블 생성에 미사용                         | 레이블 생성에 미사용                               | 필수 (Peak/Foot 검출 + 레이블)                     |
| 관련 문서     | [construct-dataset.md](construct-dataset.md) | [construct-dataset-v1.md](construct-dataset-v1.md) | [construct-dataset-v2.md](construct-dataset-v2.md) |

## 2. 전체 데이터셋 규모 비교

### 2.1 세그먼트 및 케이스 수

| 구분      | dataset                          | dataset-v1                       | dataset-v2                       |
| --------- | -------------------------------- | -------------------------------- | -------------------------------- |
| **train** | 6,873,107 segs / 2,304 cases     | 6,769,507 segs / 2,303 cases     | **5,478,776 segs / 2,307 cases** |
| **val**   | 977,299 segs / 339 cases         | 962,633 segs / 339 cases         | **777,520 segs / 339 cases**     |
| **test**  | 1,987,556 segs / 672 cases       | 1,955,049 segs / 672 cases       | **1,572,941 segs / 672 cases**   |
| **합계**  | **9,837,962 segs / 3,315 cases** | **9,687,189 segs / 3,314 cases** | **7,829,237 segs / 3,318 cases** |

> segment 기준 70/10/20 분할 비율: dataset 69.9/9.9/20.2%, v1 69.9/9.9/20.2%, v2 70.0/9.9/20.1%  
> v2는 case-level split으로 구성되므로 세그먼트 비율이 설계 목표(70/10/20)에 매우 근접한다.  
> v2 총 케이스(3,318)가 dataset(3,315)보다 약간 많은 것은 `.vital` 파일의 ABP/PPG 트랙 가용성 차이로 인한 것이다.

### 2.2 필터링 감소율

| 경로           | 세그먼트 수 | 감소 세그먼트  | 감소율     |
| -------------- | ----------- | -------------- | ---------- |
| dataset (기준) | 9,837,962   | —              | —          |
| → dataset-v1   | 9,687,189   | 150,773 제거   | **–1.5%**  |
| → dataset-v2   | 7,829,237   | 2,008,725 제거 | **–20.4%** |

dataset → v1 전환 시 power_ratio 필터는 전체 세그먼트의 1.5%만 제거한다.
dataset → v2 전환 시 9단계 ABP 기반 정제 룰은 20.4%를 제거하며, dataset-v2는 dataset의 **79.6%** 규모를 달성한다.

케이스 기준으로는 dataset-v1이 1개 케이스만 탈락하고,
dataset-v2는 dataset과 사실상 동일한 케이스 풀을 보유한다(3,318 vs 3,315).
9단계 정제 룰에서 유효 세그먼트 0개로 탈락하는 케이스가 거의 없는 수준이다.

### 3.3 주요 관찰

#### **dataset ↔ dataset-v1 비교**

두 데이터셋의 혈압 분포는 거의 동일하다. power_ratio 필터(PPG 주파수 품질)는 혈압값 자체를 기준으로 선별하지 않으므로 SBP·DBP 통계량이 사실상 변하지 않는다. 분포 그래프에서도 두 데이터셋의 곡선이 겹쳐 보인다.

#### **dataset-v2 SBP 평균: dataset/v1과 근접 (–1.3 mmHg)**

dataset-v2의 SBP 평균(116.60 mmHg)은 dataset-v1(117.87 mmHg)보다 약 1.3 mmHg 낮다.
전반적인 SBP 분포가 dataset/v1과 매우 유사한 수준이다.

이 근접성은 다음 요인들이 상쇄된 결과다:
1. **레이블 산출 방식 차이**: v2는 ABP 파형 peak에서 직접 레이블을 추출하므로 1 Hz 평균보다 약간 높은 경향이 있으나, 기여 폭이 제한적이다.
2. **룰 3 하한 적용**: SBP ≥ 60 mmHg 강제로 저혈압 구간이 제거되어 분포 하단이 절사된다.

#### **dataset-v2 SBP 범위 축소 (249.25 → 180.00 mmHg)**

룰 3(SBP ≤ 180 mmHg)이 분포 상단을 제거한다. 극단적 고혈압 세그먼트가 제외되는 것은 여전하지만,
SBP std(18.98)와 p75(128.81)는 v1(std 19.95, p75 129.75)과 거의 동일하여 중간 범위 분포는 잘 보존된다.

#### **dataset-v2 DBP 범위 축소 (20–150 → 40–120 mmHg)**

룰 3(DBP 40–120 mmHg)이 하한을 40 mmHg로 올린다. 저이완기혈압 구간이 제거되지만
DBP 전체 통계량(mean 63.90, std 11.75)은 v1(mean 63.24, std 11.81)과 사실상 동일하다.

#### **dataset-v2 SBP 단봉 분포 (bimodal 해소)**

dataset-v2 SBP 분포는 dataset/v1과 유사한 단봉 형태이며 split 간 분포 일관성도 유지된다.

## 5. 케이스 내 혈압 변동성 비교 (SD per Case)

sd_per_case.png는 케이스 내 SBP·DBP 표준편차를 케이스 단위로 정렬한 분포다.
이 지표는 교정(calibration) 없이 운용되는 혈압 추정 모델의 난이도를 반영한다.

| 통계량                      | dataset   | dataset-v1 | dataset-v2    |
| --------------------------- | --------- | ---------- | ------------- |
| SBP 케이스 내 SD (중앙값)   | 16.2 mmHg | 16.1 mmHg  | **14.5 mmHg** |
| SBP 케이스 내 SD (평균)     | 16.8 mmHg | 16.7 mmHg  | **14.8 mmHg** |
| SBP SD ≤ 5 mmHg 케이스 비율 | 0.1%      | 0.1%       | 0.1%          |
| DBP 케이스 내 SD (중앙값)   | 8.9 mmHg  | 8.9 mmHg   | **8.0 mmHg**  |
| DBP 케이스 내 SD (평균)     | 9.2 mmHg  | 9.2 mmHg   | **8.1 mmHg**  |
| DBP SD ≤ 5 mmHg 케이스 비율 | 1.4%      | 1.7%       | **5.5%**      |

dataset-v2의 케이스 내 혈압 변동성은 dataset/v1보다 약간 낮다
(SBP SD 중앙값: 14.5 vs 16.1 mmHg, DBP SD 중앙값: 8.0 vs 8.9 mmHg).
Rule 8(세그먼트 내 SBP 변동 ≤ 40 mmHg)이 혈역학적 과도기 구간을 여전히 걸러내지만,
케이스당 세그먼트 수가 충분히 많아(중앙값 2,194) 각 케이스의 전체 혈압 범위를 잘 포괄한다.
SBP SD ≤ 5 mmHg 케이스 비율이 0.1%로 dataset/v1과 동일하여, 실질적인 혈압 변동 패턴이 보존된다.

DBP SD ≤ 5 mmHg 케이스 비율이 5.5%로 dataset(1.4%)보다 약간 높은 것은 룰 3의 DBP 범위 제한(40–120 mmHg)이
혈압 변동폭을 일부 축소하는 효과에서 기인한다.


## 6. 레이블 산출 방식 비교

| 항목                 | dataset / dataset-v1                           | dataset-v2                                           |
| -------------------- | ---------------------------------------------- | ---------------------------------------------------- |
| 레이블 신호 출처     | `Solar8000/ART_SBP`, `ART_DBP` (1 Hz 수치)     | `SNUADC/ART` ABP 파형 (500 Hz → 125 Hz)              |
| SBP 산출 방법        | 세그먼트 구간 SBP 수치 평균                    | ABP peak 검출 후 `abp_raw_seg[peaks]` 평균           |
| DBP 산출 방법        | 세그먼트 구간 DBP 수치 평균                    | ABP foot 검출 후 `abp_raw_seg[foots]` 평균           |
| 시간 정합성          | 1 Hz 수치와 PPG 파형 간 약간의 시간 오프셋     | ABP·PPG 동기화된 파형에서 직접 추출 → 시간 정합 보장 |
| 레이블 값 특성       | SBP·DBP 이동 평균에 가까운 부드러운 값         | 실제 수축기·이완기 피크 진폭 → 더 높은 SBP 경향      |
| 저혈압·고혈압 극단값 | 생리 범위(SBP 50–250, DBP 20–150) 내 모두 포함 | 룰 3으로 SBP 60–180, DBP 40–120으로 제한             |

동일 케이스에서도 v2 레이블과 dataset/v1 레이블 간에 수 mmHg 차이가 발생할 수 있다.
따라서 v2로 학습한 모델의 성능을 dataset/v1 기반 모델과 직접 수치 비교할 때는 이 차이를 감안해야 한다.

## 7. 필터링 기준 비교

### 7.1 공통 기본 필터 (세 데이터셋 모두 적용)

| 조건                           | 상세                               |
| ------------------------------ | ---------------------------------- |
| NaN·Inf 포함 PPG 세그먼트 제거 | guard-band 포함 구간 전체 검사     |
| SBP 유효 비율 ≥ 50%            | Solar8000 수치 기준 (dataset/v1만) |
| DBP 유효 비율 ≥ 50%            | Solar8000 수치 기준 (dataset/v1만) |
| SBP > DBP                      | 레이블 생리적 일관성 최종 확인     |

### 7.2 dataset-v1 추가 필터

| 조건                    | 방법                               | 임계값 |
| ----------------------- | ---------------------------------- | ------ |
| PPG 심박수 대역 전력 비 | Welch PSD, 0.67–3.0 Hz / 0.5–10 Hz | ≥ 0.6  |

### 7.3 dataset-v2 추가 필터 (8개 룰, Rule 2 제외)

| 룰   | 조건                                  | 방법·임계값                                                                         |
| ---- | ------------------------------------- | ----------------------------------------------------------------------------------- |
| 룰 1 | 연속 동일값 검사 (flat line 감지)     | ABP·PPG 각각, 연속 횟수 ≤ 10 (80 ms @ 125 Hz)                                       |
| 룰 3 | 혈압 범위 제한                        | SBP 60–180 mmHg, DBP 40–120 mmHg                                                    |
| 룰 4 | 심박수 범위 제한                      | ABP·PPG 각각 30–150 bpm                                                             |
| 룰 5 | ABP–PPG 심박수 일치                   | 차이 ≤ 10 bpm                                                                       |
| 룰 6 | Peak/Foot 개수 차이                   | ABP·PPG 각각 ‖N_peak − N_foot‖ ≤ 2                                                  |
| 룰 7 | 최소 Peak/Foot 개수 (레이블 신뢰도)   | ABP peak ≥ 4, foot ≥ 4                                                              |
| 룰 8 | 세그먼트 내 혈압 변동 범위            | SBP 변동 ≤ 40 mmHg, DBP 변동 ≤ 20 mmHg                                              |
| 룰 9 | PPG Skewness (파형 방향성)            | foot-to-foot 사이클 평균 skewness > 0                                               |

> **Rule 2 비활성화**: Rule 2(FASQA 스펙트럴 품질 검사)는 과도한 필터링 문제로 `bpe/utils/qc_v2.py`에서 완전히 비활성화됐다.
> ABP와 PPG의 주파수 특성이 다름에도 동일 임계값을 적용하는 구조적 문제로 유효 세그먼트의 대부분이 탈락하여,
> 신호별 임계값 완화를 시도했으나 적절한 임계값을 결정하기 어려워 최종적으로 룰 자체를 비활성화했다.

## 9. 결론 및 활용 지침

### dataset (기본 버전)

- 가장 큰 규모(약 1,000만 세그먼트)로 모델 사전학습이나 대규모 실험에 적합하다.
- 레이블이 1 Hz 이동 평균 값이므로 세그먼트 내 혈압 변화가 평균으로 눌려 있다.
- 극단적 혈압값(저혈압·고혈압)을 포함하므로 분포 폭이 넓다.

### dataset-v1 (PPG 주파수 필터링)

- dataset에서 PPG 신호 품질이 낮은 세그먼트(1.5%)를 제거했으나 통계적 특성은 dataset과 거의 동일하다.
- power_ratio 단일 기준 필터는 ABP 품질, 생리적 타당성 등을 평가하지 않아 실질적인 개선 효과가 미미하다.
- dataset의 경량 대안으로 사용할 수 있으나 dataset-v2와 같은 강한 정제 효과는 기대하기 어렵다.

### dataset-v2 (9단계 ABP 기반 정제)

- dataset-v1의 80.8% 규모(7,829,237 세그먼트)로 충분한 학습 데이터를 확보한다.
- 케이스당 세그먼트 분포가 dataset/v1과 유사하여(중앙값 2,194, 상위 10% 비중 21.5%) 케이스 편중 문제가 없다.
- SBP·DBP 분포가 dataset/v1과 유사한 단봉 형태이며 split 간 분포 일관성도 유지된다.
- 레이블이 ABP 파형에서 직접 추출되어 시간 정합이 보장되고 레이블 신뢰도가 높다.
- 케이스 내 혈압 변동성이 dataset/v1보다 약간 낮아(SBP SD 중앙값 14.5 vs 16.1 mmHg) Rule 8의 세그먼트 단위 안정성 필터 효과가 반영된다.
- 혈압 범위가 dataset/v1 대비 제한되며(SBP 60–180, DBP 40–120 mmHg), 극단적 저혈압·고혈압 구간의 세그먼트는 포함되지 않는다.
- 레이블 품질과 데이터 규모를 동시에 확보하는 가장 균형 잡힌 선택지다.

### 모델 학습 전략 권고

1. **기본 학습**: dataset-v2를 기본 데이터셋으로 사용한다. 레이블 품질이 높고 데이터 규모도 충분하다.
2. **대규모 학습**: dataset 또는 dataset-v1도 사용 가능하다. 두 데이터셋의 통계적 특성이 거의 동일하며 규모가 가장 크다.
3. **고품질 파인튜닝**: dataset 또는 v1으로 사전학습 후 dataset-v2로 파인튜닝하면 레이블 정확도가 높은 데이터의 장점을 취할 수 있다.
4. **모델 간 공정 비교**: 서로 다른 데이터셋으로 학습한 모델을 비교할 때는 레이블 출처 차이(Solar8000 평균 vs ABP peak/foot)가 수 mmHg 오차를 유발할 수 있음을 명시해야 한다.
